# Feature Specifications

Roadmap entries are direction, not implementation authorization. Significant work needs a specification before implementation.

- `current/` contains authorized, active implementation work.
- `planned/` contains drafts that preserve direction without committing unresolved details.
- Completed specifications may be archived later.

Each specification must reference relevant ADRs and documentation, identify data and architecture effects, define validation, and use this template:

```md
# Feature Name

Status: Draft | Active | Implemented | Superseded
Last updated: YYYY-MM-DD

## Goal
## User Value
## Current Context
## User Stories
## Functional Requirements
## Interaction States
## Data Model Changes
## Architecture Impact
## Accessibility
## Error Handling
## Non-Goals
## Acceptance Criteria
## Required Tests
## Device Validation
## Documentation Updates
## Open Questions
```

An active spec authorizes only its stated scope.
